## 零、项目故事

### 这个项目是怎么来的

很多人都做过 MBTI，但做完之后，那四个字母跟你没有任何情感连接——它是一个分类，不是一个故事。

这个项目想做的事情不一样。它想让你看完结果之后说：**"这个人，说的就是我。"** 不是因为它描述了你的行为模式，而是因为它用了一个你从小就听说过的名字，用了一段几千年前就被人讲述的故事，来告诉你：你不孤单，你的性格早就有人活过了，早就有人为此付出过代价，也早就有人因此成就了不凡。

希腊神话里的神祇不是完美的——宙斯会出轨，赫拉会报复，阿喀琉斯会因为愤怒而放弃战斗，哈迪斯被人误解了几千年。他们有弱点，有执念，有过度的自尊或过度的温柔，有时候做了让人皱眉的事，却也在某个瞬间展现出令人屏息的力量。

**他们像人。这正是他们能映照我们的原因。**

所以这个测试的核心不是"你像什么类型"，而是"你的性格在神话宇宙里找到了对应的灵魂"——一个有名字、有故事、有光芒也有裂缝的存在。

### 为什么是 20 位

十二主神之外，我们加入了普罗米修斯、珀耳塞福涅、厄洛斯、尼刻、堤喀、阿喀琉斯、奥德修斯这几位——他们不都是奥林匹斯主神，但他们代表了人格图谱里那些主神无法覆盖的角落：

- 普罗米修斯：为信念付出代价的理想主义者
- 珀耳塞福涅：经历至暗之后蜕变的人
- 厄洛斯：全情投入的浪漫主义者
- 尼刻：无法接受平庸的竞争者
- 堤喀：活在当下、随机应变的乐天派
- 阿喀琉斯：燃烧式活着的极致主义者
- 奥德修斯：历经一切依然归家的坚韧者

这 20 位，覆盖了「权威 / 智慧 / 激情 / 自由 / 深沉 / 守护」六个维度的完整光谱，几乎每一种真实存在的人格倾向，都能在这里找到共鸣。

### 产品的边界

这个测试**不评判，不排名，不给结论**。

它只是一面镜子：当下的你，在这 20 个古老的灵魂里，最接近哪一个？明天你再测，可能是另一个答案，那也是真实的。

> 每个人都是流动的——当下的结果只能代表当下的状态，无法作为评判标准，无需为结果感到内耗或忧心。

## 二、用户角色

### 2.1 首次用户

第一次打开页面，没有任何本地数据。走完完整新手流程：介绍页 → 设置昵称/头像 → 测试（10题）→ Loading → 结果页。

**核心诉求：**
- 快速理解"这是什么"
- 答题过程有仪式感，不枯燥
- 结果页让人有"这说的就是我"的共鸣感，值得截图分享

### 2.2 回访用户

已完成测试，`localStorage` 中存有完整数据，再次打开时直接进入主 App（底部导航视图）。

**核心诉求：**
- 快速看到自己的结果和数据
- 能翻阅其他 19 位神祇的故事
- 修改昵称/头像，或选择重新测试

## 四、页面设计规范

### 4.1 介绍页

**目标：** 让用户在 15 秒内理解产品，产生"我想试试"的冲动。

| 区域 | 内容 |
|------|------|
| 视觉背景 | 月相符文 + 羊皮纸底色，顶部金色装饰线 |
| 主标题 | 「神话人格 · GREEK MYTH PERSONALITY」— Cinzel Decorative 字体 |
| 副文案 | 两段世界观介绍：希腊神话的宏大背景，20位神祇的存在意义 |
| 功能说明 | 3 张竖排卡片：① 测出你的神话人格 ② 阅读你的角色故事 ③ 查看人格维度数据 |
| CTA | 「开始测试 · BEGIN」主按钮 |

### 4.2 设置昵称/头像页

| 元素 | 规范 |
|------|------|
| 昵称输入 | 必填，限 12 字，placeholder：「给自己起个名字」|
| 头像选择 | 12 个 emoji 预设，点选高亮：🌟 ⚡ 🌙 🔱 🌹 🦉 🍃 🔥 🌊 💫 🗝️ 🌺 |
| UID | 系统自动生成 6 位大写字母数字，格式 `#A7F3K2`，展示在页面 |
| 进入按钮 | 未填昵称时置灰，点击无效 |

### 4.3 测试页（10 题）

| 元素 | 规范 |
|------|------|
| 顶部 | 进度条（极细 1px 线）+ 当前题号 / 10 |
| 每题结构 | 题目文字 + 4 个选项按钮 |
| 交互 | 点选后 380ms 自动进入下一题，不可返回 |
| 结束 | 最后一题作答后进入 Loading |

**题目设计原则：**
- 每题是一个真实情景，不是抽象"你倾向于…"式问法
- 4个选项各自映射不同的维度倾向，无"正确答案"
- 语气平等，无诱导，选哪个都合理

### 4.4 计算 Loading（2 秒）

- 月相旋转动效
- 文案：「命运之轮正在旋转 · THE FATES ARE WEAVING...」

### 4.5 结果页（核心页）

结果页是整个产品最重要的页面。目标：让用户有「值得截图、值得分享」的感受。

#### 神祇身份区

| 元素 | 说明 |
|------|------|
| 神祇图标 | 大尺寸 emoji，带光晕脉冲动效 |
| 神祇中文名 | 大标题，金色 |
| 神祇英文名 | 小号，字母间距宽（0.5em） |
| 身份标签 | 如「奥林匹斯之王 · 天空与雷电之神」|
| 背景水印 | 同款 emoji 超大尺寸，低透明度衬底 |

#### 特质标签

5 个 tag，金色线框样式，如「领袖 / 权威 / 果断 / 骄傲 / 慷慨」。

#### 性格解读（约 150 字）

第二人称（"你"），让用户感到共鸣。包含：
- 核心性格特征
- 行事风格与思维模式
- 潜在盲点或阴影面
- 与他人相处的方式

#### 神话故事摘要（约 150 字）

该神祇最具代表性的故事，有画面感。包含：
- 故事背景与起因
- 最关键的转折或行动
- 故事与其人格的内在联系

#### 人格维度数据（6维）

横向条形图，动画从 0 填充至实际值，右侧显示百分比。

| 维度 | 含义 |
|------|------|
| 权威 | 领导力、掌控欲、秩序感 |
| 智慧 | 逻辑、洞察、策略思维 |
| 激情 | 情绪强度、行动力、感染力 |
| 自由 | 独立性、反传统、对束缚的抗拒 |
| 深沉 | 内省、神秘感、情感的深度 |
| 守护 | 同理心、牺牲精神、对他人的关怀 |

#### 经典名言

斜体引用框，左侧金色竖线，1-2 句话，代表该神祇的核心世界观。

#### 命运羁绊

展示当前匹配神祇与 6 位神祇的关系，每种关系各取 1 位，形成以「你」为中心的神话关系图谱。

**关系类型（共 6 种，各取 1 位）：**

| 类型 | 取法 | 本质说明 |
|------|------|------|
| 🔮 命途相镜 | 余弦相似度最高 | 灵魂镜像，你们是同一类人 |
| 🌑 暗涌之缘 | 余弦相似度第 2 高 | 有深度联结，但都藏着不说 |
| 💘 幻想之人 | 余弦相似度第 5 位 | 被彼此吸引，却始终有距离 |
| 🗝️ 解惑之神 | 余弦相似度第 15 位 | 你缺的，恰好是他最多的 |
| 🌊 过客如风 | 余弦相似度第 18 位 | 交汇一瞬，各自走向远方 |
| ⚔️ 宿命之敌 | 余弦相似度最低 | 最根本的性格碰撞，势不两立 |

**呈现形式：**

每条羁绊展示：神祇头像 + 名字 + **关系类型标签**（放在最显眼位置）+ 一句关系描述文案（15字以内）。

关系类型标签是核心视觉焦点——用户看到「幻想之人」「宿命之敌」时会立刻想点开，想截图分享。

**关系标签样式：**
- 命途相镜：金色，最暖
- 暗涌之缘：深紫棕，神秘
- 幻想之人：玫瑰金，梦幻
- 解惑之神：橄榄金，智慧
- 过客如风：灰棕，轻盈
- 宿命之敌：深炭，凌厉

**数据来源：** 复用 6 维人格数值做余弦相似度计算，无需新增字段。关系描述文案沿用 CHARACTERS.md 中已有的每对关系文案。

#### 操作区

- **分享结果**：复制当前页面 URL
- **再测一次**：清除测试数据，回到介绍页（保留昵称/头像）

## 六、角色数据规范

### 6.1 字段标准

| 字段 | 字数要求 | 写作视角 |
|------|---------|---------|
| 身份标签 | 15 字以内 | 三人称，神话定位 |
| 特质标签 | 5个，每个 2-4 字 | 名词/形容词 |
| 性格解读 | 120-160 字 | **第二人称**（"你"） |
| 神话故事摘要 | 120-160 字 | 三人称，重画面感 |
| 经典名言 | 20-40 字 | 第一人称，该神祇口吻 |
| 6维人格数值 | 0-100 整数 | — |

### 6.2 写作原则

**性格解读的核心要求：**
- 第一句话必须是"你有…"或"你是那种…"，立刻建立共鸣
- 必须包含一个"弱点"或"阴影面"——没有阴影的角色是虚假的
- 不用"你喜欢"而用"你天生"——前者是偏好，后者是性格
- 结尾一句要有余味，不要总结性陈述，要留白

**神话故事摘要的核心要求：**
- 有画面感，读者能在脑海中看见场景
- 最后一两句揭示故事与人格的内在联系
- 不要流水账式复述，要抓住最戏剧性的一个瞬间

## 八、数据结构

### localStorage 键

| Key | 类型 | 生命周期 |
|-----|------|---------|
| `myth_user` | JSON | 持久，直到用户主动清除 |

### myth_user 数据结构

```json
{
  "nickname": "晓明",
  "avatar": "🌟",
  "uid": "A7F3K2",
  "godId": "zeus",
  "dims": {
    "权威": 85,
    "智慧": 60,
    "激情": 70,
    "自由": 40,
    "深沉": 45,
    "守护": 55
  },
  "testedAt": "2026-06-10T12:00:00Z"
}
```

### 匹配算法原则

用户 10 题的选项各自映射 6 维的得分，累加后归一化为 0-100。将用户的 6 维向量与 20 位神祇的标准向量做余弦相似度计算，取相似度最高的一位作为匹配结果。

## 十、设计规范

设计系统文件位于 `.claude/skills/greek-myth-design/`，完整规范见 `DESIGN.md`。

**核心设计原则：**
- 背景：羊皮纸奶油色 `#F5F0E8`，永远不用纯白
- 强调：只有深金 `#B8860B` 一种颜色，不引入其他彩色
- 线框：所有卡片和按钮 0px 圆角，无阴影，层次靠描边表达
- 字体：Cinzel Decorative（神祇名大标题）· Cinzel（标题/按钮）· Noto Serif SC（正文）
- 装饰：Unicode 符号（✦ ◆ ☽ ⊱ ⊰）替代图标库
